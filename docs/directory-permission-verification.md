# ディレクトリ選択の権限拒否・実機動作検証

「フォルダを開いて監視」(ウォッチモード起動)は File System Access API の `showDirectoryPicker` を使う。
ブラウザ/OSがフォルダへのアクセスを拒否した場合(ユーザーがダイアログで拒否した、サイト設定でファイルシステム
アクセスがブロックされている等)、[App.tsx](../apps/web/src/App.tsx) の `handleOpenDirectory` が
例外を捕捉し、エラーバナーを表示してアプリの他の機能(ドラッグ&ドロップでの手動読み込み等)を継続利用できる
状態を保つ。この一連の権限拒否ハンドリングは自動テスト
([tests/App.directoryError.test.tsx](../apps/web/tests/App.directoryError.test.tsx))で
`showDirectoryPicker`・フォルダ読み込み中の双方の `SecurityError` をフェイクで注入して検証しているが、
実ブラウザのダイアログ・OS権限設定を介した経路は自動テストの対象外のため、本書の手順で手動確認する。

## 前提

- **ブラウザ**: File System Access API 対応の Chrome または Edge(Chromium 系)。Firefox / Safari では
  「フォルダを開いて監視」ボタン自体が表示されないため、本検証の対象外。
- **サーバー**: 開発サーバーを起動しておく。

  ```bash
  docker compose up -d web    # http://localhost:5173
  # または
  npm install && npm run -w @model-checking/web dev
  ```

## エラーメッセージの対応

| 状況 | `handleOpenDirectory` の分岐 | 表示されるメッセージ |
| --- | --- | --- |
| `showDirectoryPicker` 呼び出し自体または読み込み中に `DOMException("SecurityError")` が発生 | `error instanceof DOMException && error.name === "SecurityError"` | フォルダへのアクセスが拒否されました |
| それ以外の例外(読み込み失敗など) | 上記以外 | フォルダの読み込みに失敗しました: `{エラーメッセージ}` |
| ユーザーがピッカーをキャンセルした(`AbortError`) | [fsaWatchTarget.ts](../apps/web/src/ui/fsaWatchTarget.ts) の `pickDirectoryWatchTarget` が `null` を返す | 何も表示しない(エラー扱いしない) |

いずれのエラーの場合も、エラー表示後は `openingDirectory` が解除され「フォルダを開いて監視」ボタンが再び押せる
状態に戻る。監視状態(`dirName`)には入らない。

## 検証手順

各項目の「期待」をすべて満たせば合格。

### 1. フォルダ選択ダイアログでの拒否

1. アプリを開く(`http://localhost:5173`)。
2. 「ウォッチモード」セクションの **「フォルダを開いて監視」** をクリックする。
3. OS/ブラウザのフォルダ選択ダイアログで、閲覧許可のプロンプトが出た場合に **「許可しない」** を選ぶ
   (ブラウザによってはダイアログ自体をキャンセルするのと権限を明示的に拒否するのが別操作になる。
   後者の権限拒否の操作を行うこと)。

**期待**:
- 「フォルダへのアクセスが拒否されました」というエラーバナー(見出し「フォルダの読み込みエラー」)が
  表示される。アプリはクラッシュしない。
- 「フォルダを開いて監視」ボタンが再び押せる状態に戻っている(「監視中」表示には入らない)。

### 2. サイト設定でファイルシステムアクセスをブロックした状態での選択

1. ブラウザのサイト設定(Chrome: アドレスバー左のアイコン → サイトの設定、または
   `chrome://settings/content/siteDetails?site=http://localhost:5173`)で、
   「ファイルの編集」等ファイルシステムアクセスに関する権限を **「ブロック」** に変更する。
2. ページを再読み込みし、「フォルダを開いて監視」をクリックしてフォルダを選ぶ。

**期待**:
- 手順1と同様に「フォルダへのアクセスが拒否されました」バナーが表示される
  (ブラウザによっては読み込み処理の途中で拒否が起き、この場合も同じバナーになる)。
- 手順後、サイト設定を元(許可)に戻しておく。

### 3. フォールバック(手動読み込み)の確認

1. 手順1または2でエラーバナーが表示された状態から続ける。
2. 画面上部のドラッグ&ドロップ領域(DropZone)、または「ファイルを選択」ボタン、もしくは
   ワンクリックデモボタン(`payment-retry` 等)を使ってファイルを読み込む。

**期待**:
- エラーバナーが出ていても、DropZoneや通常のファイル選択・デモ読み込みは問題なく動作し、
  解析・検査に進める。
- 新しいファイルを読み込むと、直前のディレクトリ関連のエラーバナーとは独立に通常の画面に遷移する
  (エラー状態が後続の操作をブロックしない)。

## 記録

検証時は以下を控える(不具合報告時の再現条件になる)。

- ブラウザ名とバージョン(例: Chrome 126 / Edge 126)
- OS
- 各手順の合否
- 不合格があれば、操作と実際の挙動・スクリーンショット
